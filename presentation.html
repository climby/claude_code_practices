PRD 驱动开发

第一步不是写代码，而是和 Claude Code 一起完善 PRD

PRD 包含什么

  • 技术栈选型 + 选择理由
  • 数据模型（表结构定义）
  • API 契约（路径、参数、权限）
  • UI 交互规范（布局、颜色、防误操作）
  • 开发环境与部署方式

为什么有效

  • 给 AI 明确的边界和约束
  • 可分阶段执行，每步目标清晰
  • PRD 本身成为项目文档

构建

#!/bin/bash
rsync -avz --delete \
  --exclude '.git' --exclude 'node_modules' \
  ./ maodf@nuc2:/home/maodf/sandbox/zpod/
ssh maodf@nuc2 "cd /home/maodf/sandbox/zpod && docker compose up -d --build"

Claude Code 直接执行 sync_and_run.sh，看到构建输出和报错，形成快速迭代闭环

/btw — 不打断工作流的灵感捕捉

场景

AI 正在帮你实现功能 A，你突然想到功能 B 的改进

传统做法

等 A 做完再说，或打断当前任务 → 上下文混乱

使用 /btw

随时插入想法，AI 记住并在合适时机处理

实际体验

开发 ZPod 过程中，在实现后端接口时：

/btw 前端 Kill 按钮在 PROD
环境下应该用红色确认框

AI 继续完成当前接口开发，到前端阶段时自动应用了这个想法

/rewind — 精确回退的后悔药

是什么

回溯到对话中的某个节点，撤销之后的所有文件更改

  • 比 git checkout 更精确 — 它知道哪些文件改过
  • 即使没有 commit，也能精确回退
  • 回退的是对话中的操作步骤，不是 git commit

使用场景

让 AI 重构了一段代码，结果跑不通

/rewind 一键回到重构前

AI 修改了多个文件，方向不对

/rewind 全部还原，换个思路重来

! Bash Mode — 随时切换到终端

是什么

在 Claude Code 提示符中输入 ! 前缀，直接执行 shell 命令

  • 无需退出对话，随时切换到终端
  • 命令输出直接进入对话上下文
  • 适合快速查看文件、检查状态、运行脚本

! ls   ! git status   ! docker ps

输入 ! 前缀即可执行 shell 命令
